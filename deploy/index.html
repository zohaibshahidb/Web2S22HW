Project Folder vs. Root Folder

Lets take a look at the difference between our Project Folder verses our Root Folder.

Project Folder

Josh-web-2

For this class your project folder will always be the folder with "your name"-Web2

Root Folder

Deploy

The Root Folder will be found inside the Project Folder. Usually you will have a Asset and Design folder on the same level as the Root Folder which we will label Deploy.

File Conversions

.kit

Location:

Dev

All of your .kit files will be created and placed in the Dev folder. CodeKit will convert your .kit files into .html files and place them into the Deploy folder for you.

.scss

Location:

Dev

scss

Any stylingthat you write in this class should be inside one of your SCSS partial files. (We will talk more about partials in a later lecture). All of the SCSS code you write will be converted into CSS code and placed into your styles.css file by CodeKit.

.js

Location:

Dev

js

You will not have to write any Javascript code for this class. Any JS code will be given to you and we will show you were to put it.

.html

Location:

Deploy

The Deploy folder will contain all of your .html files. These are also the files that you will push to your web server via an FTP application like Cyber Duck.

.css

Location:

Deploy

css

You should never make any style changes directly to the styles.css file. If you want to make style changes you should do so by updated the partial SCSS file and compile that file.

.min.js

Location:

Deploy

js

CodeKit will take our Javascript code and minify and place it into the js folder inside the Deploy folder for us.
